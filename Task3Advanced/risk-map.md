# Карта рисков трансформации — «Будущее 2.0»

Риски сгруппированы по трём категориям: архитектурные, организационные, технологические.
Для каждого риска определены вероятность (В/С/Н) и влияние на бизнес (В/С/Н).

---

## Реестр рисков

### Архитектурные риски

| ID  | Риск | Вероятность | Влияние |
|-----|------|:-----------:|:-------:|
| A01 | Сложность миграции монолитного DWH (SQL Server 2008): встроенная бизнес-логика, перекрёстные зависимости, отсутствие документации | Высокая | Высокое |
| A02 | Деградация качества и целостности данных при CDC-миграции из DWH в доменные БД | Средняя | Высокое |
| A03 | ESB (Camel) остаётся узким местом и точкой отказа на весь период переходного состояния | Высокая | Среднее |
| A04 | Несовместимость и дрейф схем событий между доменами при эволюции контрактов | Средняя | Среднее |
| A05 | Проблема двойной записи: риск рассинхронизации данных при параллельной работе Legacy DWH и новых доменных БД | Низкая | Высокое |

### Зоны риска

| Зона | Риски | Приоритет |
|------|-------|-----------|
| 🔴 Критическая (высокая вероятность + высокое влияние) | A01, O01, O02 | Немедленные меры |
| 🟠 Значительная (средняя вероятность + высокое / высокая вероятность + среднее) | A02, A03, O04, T01, T02 | Плановые меры до старта этапа |
| 🟡 Умеренная | A04, A05, O03, O05, T03, T04 | Мониторинг + контрмеры |
| 🟢 Низкая | T05 | Наблюдение |
