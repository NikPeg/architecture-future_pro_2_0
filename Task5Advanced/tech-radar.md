# Технический радар — «Будущее 2.0»

Радар охватывает технологии и архитектурные паттерны целевого ландшафта на горизонте 3 лет.
Пересмотр: раз в 6 месяцев (платформы и инструменты), раз в год (методы и паттерны).

**Кольца:**
- **Adopt** — применять повсеместно, низкий риск, проверено в production
- **Trial** — опробовано на пилоте, рекомендуется для новых проектов с осторожностью
- **Assess** — изучить, создать прототип, накопить экспертизу
- **Hold** — не использовать в новых проектах, вывести из эксплуатации при возможности

## Методы и архитектурные паттерны

| Технология / Паттерн | Статус | Тренд | Обоснование |
|----------------------|--------|-------|-------------|
| **Event-Driven Architecture** | Adopt | ↑ Новое | Целевая парадигма взаимодействия доменов. Заменяет синхронные вызовы через ESB. Ключевое архитектурное решение. |
| **Domain-Driven Design (DDD)** | Adopt | ↑ Новое | Основа разбивки на bounded contexts. Применяется при проектировании всех новых сервисов и Data Mesh-доменов. |
| **Data Mesh** | Adopt | ↑ Новое | Принят как целевой подход к владению данными. Домены отвечают за свои Data Products. Self-service платформа строится под эту модель. |
| **Self-service BI** | Adopt | ↑ Новое | Бизнес-пользователи строят отчёты самостоятельно через DataLens без зависимости от IT. Снижает нагрузку на аналитиков. |
| **Data Lakehouse** | Trial | ↑ Новое | Bronze/Silver/Gold архитектура поверх Object Storage + Delta Lake. Проверена на пилоте Финтех-домена. Расширяется на все домены. |
| **Strangler Fig Pattern** | Adopt | ↑ Новое | Основной паттерн вывода Legacy DWH и ESB. Новые домены работают через Event Bus, Legacy остаётся read-only до полной миграции. |
| **API-first Design** | Trial | → Без изм. | Все новые сервисы проектируются API-first с OpenAPI-спецификацией. Внедряется через ADR-процесс. |
| **MLOps** | Trial | ↑ Новое | Применяется для управления жизненным циклом AI-моделей. Пилот на ИИ-домене. Нужен для промышленного масштабирования диагностических моделей. |
| **CQRS** | Assess | ↑ Новое | Изучается для сервисов с высокой read/write асимметрией (Fintech, MedTech телеметрия). Создан прототип. |
| **GraphQL Federation** | Assess | ↑ Новое | Оценивается как альтернатива REST для кросс-доменных read-запросов от Self-service BI. Пока не принято окончательное решение. |
| **Monolithic Integration (ESB-style)** | Hold | ↓ Устаревает | Apache Camel как центральный оркестратор. Не применять в новых интеграциях. Вывести до 18-го месяца. |
| **ETL в DWH (хранимые процедуры)** | Hold | ↓ Устаревает | Бизнес-логика в SQL Server. Запрещена к расширению. Выносить в доменные сервисы при миграции. |

## Платформы

| Технология | Статус | Тренд | Обоснование |
|------------|--------|-------|-------------|
| **Yandex Cloud** | Adopt | ↑ Новое | Основная облачная платформа. Соответствует требованиям 152-ФЗ (ЦОД в РФ). Managed-сервисы снижают операционную нагрузку. |
| **Yandex Object Storage** | Adopt | ↑ Новое | S3-совместимое хранилище для Bronze/Silver/Gold слоёв Lakehouse. Основа Data Platform. |
| **Yandex Managed Kafka** | Adopt | ↑ Новое | Managed Kafka для Event Bus. Снимает операционную нагрузку по обслуживанию кластера. |
| **Yandex Managed PostgreSQL** | Adopt | ↑ Новое | Доменные БД для каждого bounded context. Автоматические бэкапы, failover, мониторинг. |
| **Yandex Kubernetes (YC MK8S)** | Adopt | ↑ Новое | Оркестрация всех микросервисов доменов. Уже используется в Fintech-домене. |
| **Yandex DataLens** | Adopt | ↑ Новое | Self-service BI-платформа. Нативная интеграция с Yandex Cloud, RBAC, бесплатный базовый тариф. |
| **Apache Spark (Yandex DataProc)** | Trial | ↑ Новое | Обработка данных между слоями Lakehouse. Managed через Yandex DataProc. Пилот завершён успешно. |
| **Yandex DataSphere** | Trial | ↑ Новое | MLOps-платформа для ИИ-домена. Обучение и деплой ML-моделей. Интеграция с YC-инфраструктурой. |
| **Apache Airflow (Managed)** | Trial | ↑ Новое | Оркестрация Spark-пайплайнов. Используется на пилоте. До принятия окончательного решения оценивается Yandex Managed Airflow. |
| **SQL Server 2008 (on-prem)** | Hold | ↓ Устаревает | EOL с 2019 года. Нет патчей безопасности. Только read-only ACL до завершения миграции. |
| **PowerBuilder** | Hold | ↓ Устаревает | Клиентский интерфейс операторов. Вывод из эксплуатации в первые 12 месяцев. |

## Инструменты

| Технология | Статус | Тренд | Обоснование |
|------------|--------|-------|-------------|
| **Confluent Schema Registry** | Adopt | ↑ Новое | Централизованное хранение и валидация схем Kafka-событий (Avro). CI/CD-проверка контрактов при деплое. |
| **DataHub** | Trial | ↑ Новое | Data Catalog: поиск, lineage, документация, RBAC. Пилот развёрнут. Решение принято, расширяется на все домены. |
| **Delta Lake** | Trial | ↑ Новое | ACID-транзакции поверх Object Storage в Lakehouse. Версионирование данных, time travel. Пилот на Fintech-домене. |
| **Great Expectations / Deequ** | Trial | ↑ Новое | Data quality gates между слоями Lakehouse. Пилотируется. Принять решение о стандарте до этапа масштабирования. |
| **Terraform** | Adopt | → Без изм. | IaC для всей облачной инфраструктуры. Уже применяется (Task1-2). |
| **GitHub Actions** | Adopt | → Без изм. | CI/CD для инфраструктуры и сервисов. Уже применяется. |
| **Apache Iceberg** | Assess | ↑ Новое | Альтернатива Delta Lake для table format. Оценивается на предмет лучшей поддержки мультиоблачных сценариев. Решение — к 18-му месяцу. |
| **Power BI** | Hold | ↓ Устаревает | Заменяется Yandex DataLens. Не использовать для новых отчётов. Мигрировать критичные дашборды до конца Year 2. |
| **Apache Camel ESB** | Hold | ↓ Устаревает | Только как Compatibility Adapter на переходный период. Вывести к 18-му месяцу. |
